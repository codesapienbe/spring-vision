# Fix Embedding Extraction Failure

## Root Cause Analysis

The error logs reveal the following flow:

1. **ONNX Model Missing**: `RetinaFaceDetector` logs `onnx.missing` warning because `retinaFaceOnnxPath()` is null or blank (line 49 in `RetinaFaceDetector.java`)
2. **Fallback to OpenCV**: When ONNX path is missing, `detectFaces()` calls `fallbackOpenCv().detectFaces()` (line 100)
3. **Empty Detections**: Face detection returns empty list, likely due to configuration or detector issues
4. **Empty Embeddings**: `DeepFace.represent()` returns `Collections.emptyList()` when no faces detected (line 1010 in `DeepFace.java`)
5. **Null Error Message**: `VisionTool.extractEmbeddings()` catches exception with `e.getMessage()` which is null, producing the message "Failed to extract embeddings: null"

## Solution Strategy

### 1. Fix Null Error Messages (VisionTool.java)

- Handle cases where exception message is null
- Log full stack trace for debugging
- Return more descriptive error messages

### 2. Improve Null Handling in EmbeddingSupport.java

- Add proper null checks and meaningful error messages

### 3. Handle Empty Detection Results Gracefully

- Ensure empty embedding lists are properly handled without null exceptions
- Add logging to identify why detections are empty

### 4. Improve Error Propagation

- Ensure exceptions with null messages are caught and wrapped with context
